### TODO 1: EPUB Library Evaluation Spike
**Priority:** High (blocks export module)
**Effort:** human ~2 days / CC ~20 min

### TODO 3: White-Label Theming Spec
**Priority:** Medium (blocks Settings UI)
**Effort:** human ~1 day / CC ~15 min

Design the per-key white-label customization system:

**Firestore side:**
- `can_whitelabel` boolean flag on key record
- Admin toggle in web app

**Client side (Settings page):**
- Logo upload (PNG/SVG, stored locally in app data dir)
- Color extraction: dominant color from logo using canvas pixel sampling
- Light/dark mode decision based on logo luminance (not system preference)
- CSS variable mapping: `--brand-primary`, `--brand-bg`, `--brand-text`
- App name override (optional, from key metadata)

**Deliverable:** Spec document with Firestore schema additions, Settings UI wireframe, color algorithm pseudocode.

### TODO 4: Admin Web App Data Model
**Priority:** Medium (blocks credit management)
**Effort:** human ~1 day / CC ~15 min

Define the Firestore collections and admin workflows:

**Collections:**
- `keys` — activation key, client label, credit balance, `can_whitelabel`, status (active/revoked/expired), created_at, expires_at
- `usage_logs` — key_id, job_type (ocr/extract/translate/wizard), pages_processed, credits_charged, timestamp, status (success/partial/failed)

**Workflows:**
- Key provisioning: generate key, set initial credits, set flags
- Credit top-up: add credits to existing key, log the top-up
- Usage monitoring: per-key usage over time, aggregate dashboard
- Key revocation: soft-delete, block all processing

**Deliverable:** Firestore schema document + admin UI wireframe sketches.
